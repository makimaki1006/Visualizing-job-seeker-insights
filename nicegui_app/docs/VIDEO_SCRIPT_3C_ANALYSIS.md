# 採用戦略における3C分析と求職者市場調査の重要性

**なぜ「市場に求職者がどれだけいるか」を調べることが重要なのか**

---

## 第1章: 採用における3C分析とは

### 1.1 3C分析の基本フレームワーク

マーケティングの基本フレームワーク「3C分析」は、採用活動にも適用できます。

```
┌─────────────────────────────────────────────────────────┐
│                    採用における3C分析                    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│    ┌───────────┐                                        │
│    │ Customer  │  ← 求職者（採用したい人材）            │
│    │  求職者   │    「市場にどれだけいるのか？」        │
│    └─────┬─────┘                                        │
│          │                                              │
│    ┌─────┴─────┐                                        │
│    │           │                                        │
│  ┌─┴───────┐ ┌─┴───────┐                               │
│  │Competitor│ │ Company │                               │
│  │  競合他社 │ │  自社   │                               │
│  └─────────┘ └─────────┘                               │
│   「他社は何人  「自社の採用                             │
│    採用する？」   条件は適切？」                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

| 要素 | 採用活動での意味 | 問いかけ |
|------|-----------------|---------|
| **Customer（求職者）** | 採用したい人材 | 市場にどれだけいるのか？ |
| **Competitor（競合）** | 同じ人材を採用したい他社 | 競合は何人採用しようとしている？ |
| **Company（自社）** | 採用する側の企業・施設 | 自社の条件は市場に合っている？ |

## 第2章: なぜ「求職者市場」を調べることが最も重要なのか

### 2.1 従来の採用活動の致命的な問題

多くの企業が陥る「盲目的採用」の実態：

```
【従来の採用フロー】

  「介護福祉士を採用したい」
           ↓
  求人媒体に掲載 ← ここでいきなり行動
           ↓
  応募が来ない...
           ↓
  「なぜ？」 ← 原因が分からない
           ↓
  掲載費を追加投入
           ↓
  また応募が来ない...
           ↓
  「人手不足だから仕方ない」 ← 思考停止
```

**問題点**: そもそも「市場に求職者がいるのか」を確認していない

### 2.2 商品販売との比較

商品販売では当たり前のことが、採用では行われていない：

| 比較項目 | 商品販売 | 採用活動（従来） |
|---------|---------|-----------------|
| 市場調査 | 必ず行う | ほぼ行わない |
| 顧客数の把握 | 「見込み客は○万人」 | 「分からない」 |
| 競合分析 | 「競合シェアは○%」 | 「分からない」 |
| 価格設定 | 市場相場を調査 | 「うちはこの給与」 |
| 投資判断 | データに基づく | 感覚と経験 |

**結論**: 採用活動は「マーケティング不在」の状態で行われている

## 第3章: 求職者市場を調べないことの代償

### 3.1 実際に起こる問題

#### ケース1: 存在しない人材を探し続ける

```
【失敗例】
条件: 30代・男性・介護福祉士・正社員希望・年収400万円以上

現実: 該当者は市場に45人しかいない（全体の0.03%）

結果:
- 6ヶ月間求人を出し続ける → 掲載費100万円
- 応募ゼロ
- 「人手不足だから仕方ない」と諦める

本当の原因: そもそも存在しない人材を探していた
```

#### ケース2: 競合が多すぎる市場で戦う

```
【失敗例】
ターゲット: 東京都23区・20代・看護師

現実:
- 該当求職者: 500人
- 同エリアの求人数: 3,000件
- 競争倍率: 6倍（1人の求職者に6社が殺到）

結果:
- 応募は来るが、内定辞退が続出
- 採用単価が高騰

本当の原因: 競合過多の市場で戦っていた
```

#### ケース3: 隣の市に人材がいることを知らない

```
【失敗例】
採用エリア: A市のみ

現実:
- A市の求職者: 50人
- 隣のB市の求職者: 200人
- B市からA市への移動希望者: 68人（34%）

結果:
- A市だけで採用活動 → 苦戦
- B市の人材を見逃す

本当の原因: 移動パターンを把握していなかった
```

### 3.2 代償の定量化

| 問題 | 金銭的損失 | 時間的損失 |
|------|-----------|-----------|
| 存在しない人材を探す | 掲載費50-200万円/年 | 6-12ヶ月 |
| 競合過多市場で戦う | 採用単価2-3倍 | 内定辞退で振り出し |
| 隣接エリアを見逃す | 機会損失（採用できたはずの人材） | 慢性的な人手不足 |

## 第4章: 3C分析における「Customer（求職者）」の調べ方

### 4.1 調べるべき5つの項目

| # | 調査項目 | 問いかけ | 得られる示唆 |
|---|---------|---------|-------------|
| 1 | **総数** | 市場に何人いるのか？ | 採用の実現可能性 |
| 2 | **属性分布** | 年齢・性別・資格の構成は？ | ターゲット設定の妥当性 |
| 3 | **希少性** | 希望条件に合う人は何人？ | 条件緩和の必要性 |
| 4 | **移動意向** | 他エリアから来てくれるか？ | 採用エリアの拡大可能性 |
| 5 | **競合状況** | 同じ人材を狙う競合は？ | 差別化ポイント |

### 4.2 このツールで調べられること

```
【MapComplete ダッシュボードで確認できる情報】

1. 総数
   → 「東京都には132,690人の求職者がいます」

2. 属性分布
   → 「50代女性が最多（19.6%）、40代女性が次点（15.2%）」

3. 希少性（レアリティ分析）
   → 「30代・男性・介護福祉士は45人（0.03%）= 超希少」

4. 移動意向
   → 「49%が地元志向、34%が近隣移動可、17%が広域移動可」

5. 競合状況（需給バランス）
   → 「この地域は供給過多（求職者 > 求人）= 採用しやすい」
```

## 第5章: 具体的シナリオ（ストーリー形式）

### シナリオ1: 「採用できない」と思っていた施設長の発見

```
【登場人物】
田中さん: 介護施設の施設長（採用に苦戦中）

【Before】
田中さん「うちは地方だから人が来ない。介護福祉士なんて全然採用できない」
→ 2年間、同じ求人を出し続けている
→ 年間掲載費: 150万円
→ 採用実績: 0人

【ツール導入後】
田中さん「えっ、うちの市に介護福祉士希望者が200人もいるの？」

データ確認:
- 自市の求職者: 200人
- うち介護福祉士保有者: 70人
- 50代女性が6割

田中さん「なぜ応募が来ないんだ...？」

レアリティ分析で自社条件を入力:
- 30代以下
- 正社員
- 夜勤可能

結果: 該当者 3人

田中さん「3人しかいないのに、この条件で求人出してたのか...」

条件を変更:
- 50代まで可
- パート可
- 日勤のみ可

結果: 該当者 85人

【After】
田中さん「条件を変えたら、2週間で5人応募が来た」
→ 採用成功
→ 「人がいない」のではなく「条件が市場と合っていなかった」
```

### シナリオ3: 採用コンサルタントの提案

```
【登場人物】
佐藤さん: 採用コンサルタント
クライアント: 訪問介護事業所

【クライアントの悩み】
「介護福祉士を5名採用したいが、半年間応募がない」

【佐藤さんの分析】

Step 1: 現状の採用条件を確認
- 年齢: 35歳以下
- 資格: 介護福祉士必須
- 雇用形態: 正社員
- 勤務地: 渋谷区

Step 2: レアリティ分析で該当者を確認
結果: 12人（全体の0.1%）

佐藤さん「この条件に合う人材は、渋谷区に12人しかいません」

クライアント「えっ、そんなに少ないの？」

Step 3: 条件緩和シミュレーション

| 条件変更 | 該当者数 | 増加率 |
|---------|---------|-------|
| 現状（35歳以下） | 12人 | - |
| 45歳以下に変更 | 58人 | 4.8倍 |
| 実務者研修も可 | 120人 | 10倍 |
| パートも可 | 200人 | 16.7倍 |

佐藤さん「年齢上限を45歳に広げるだけで、候補者が4.8倍になります」

クライアント「そうか、条件が厳しすぎたのか...」

Step 4: 移動パターン分析

佐藤さん「さらに、世田谷区・目黒区からの流入希望者が34%います。
採用エリアを広げれば、さらに母集団が拡大します」

【提案内容】
1. 年齢上限を45歳に緩和 → 候補者4.8倍
2. 実務者研修も対象に → 候補者10倍
3. 世田谷区・目黒区も採用エリアに → さらに拡大

【結果】
条件変更後、1ヶ月で8名の応募、3名の採用に成功
```

## 第6章: 3C分析における求職者調査の位置づけ

### 6.1 3C分析の順序

```
【正しい3C分析の順序】

Step 1: Customer（求職者）を調べる ← まずここから
        「市場にどれだけいるのか？」
        「どんな属性の人が多いのか？」
        「自社の条件に合う人は何人いるのか？」
               ↓
Step 2: Competitor（競合）を調べる
        「同じ人材を狙っている競合は何社？」
        「競合の採用条件は？」
        「需給バランスは？」
               ↓
Step 3: Company（自社）を見直す
        「自社の条件は市場に合っているか？」
        「差別化ポイントは何か？」
        「条件緩和すべきか？」
               ↓
Step 4: 戦略立案
        「どのターゲットに、何を訴求するか」
```

### 6.2 Customer調査が最初である理由

```
【Customer調査を省略した場合】

❌ 悪い例:
「競合が○○の条件を出しているから、うちも」
→ そもそも市場に人がいなければ、条件を上げても無意味

❌ 悪い例:
「自社の予算は○○だから、この条件で」
→ 市場に合っていなければ、応募は来ない

✅ 良い例:
「市場に○○人いて、△△の属性が多い。
 だから自社条件を□□に設定し、
 競合との差別化として◇◇を訴求する」
→ データに基づいた戦略
```

## 第7章: まとめ

### 7.1 求職者市場を調べることの3つの価値

| # | 価値 | 具体的メリット |
|---|------|---------------|
| 1 | **実現可能性の判断** | 採用目標が達成可能か、事前に分かる |
| 2 | **条件設定の最適化** | 市場に合った条件で、応募率向上 |
| 3 | **投資判断の根拠** | 「なぜこの予算か」をデータで説明できる |

### 7.2 このツールが提供する価値

```
【従来】
「採用したい」→「求人を出す」→「待つ」→「来ない」→「なぜ？」

【このツール導入後】
「採用したい」→「市場を調べる」→「戦略を立てる」→「求人を出す」→「採用成功」
                    ↑
              MapComplete ダッシュボード
```

### 7.3 最後のメッセージ

```
「人手不足」は本当か？

多くの場合、
- 人がいないのではなく、「どこにいるか分からない」
- 応募が来ないのではなく、「条件が市場と合っていない」
- 採用できないのではなく、「競合過多の市場で戦っている」

答えはデータの中にある。
まず、市場を調べることから始めよう。
```
